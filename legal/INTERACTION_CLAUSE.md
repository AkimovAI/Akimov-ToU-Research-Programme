# Interaction Clause

Author: Ing. Alex Akimov

This clause defines how interaction with the ToU Engine and related systems is legally interpreted.

For the purposes of this project, any interaction with the ToU Engine, its components, or derived systems through:

- APIs
- RPC interfaces
- shared memory
- network sockets
- inter-process communication
- or any form of programmatic interface

may constitute the creation of a combined or derivative system, depending on the level of functional integration and dependency.

If such interaction forms a system where the ToU Engine provides essential functionality, logic, or computation, the resulting system may be subject to the same licensing requirements as the original framework.

In such cases:

- The system must comply with the applicable open-source license (e.g. AGPLv3), or
- Obtain a commercial or custom license from the author

This clause ensures that remote usage does not bypass licensing obligations.
