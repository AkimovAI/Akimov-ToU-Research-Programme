# License Summary – Akimov Theory of Unity (ToU)

Author: Ing. Alex Akimov

This document provides a structured overview of the licensing framework governing the Akimov Theory of Unity (ToU) project and its components.

---

## 1. Core Principle

All components of the ToU ecosystem are protected under a unified licensing philosophy defined by the Akimov License Framework (ALF).

The licensing model is designed to:
- preserve authorship
- ensure ethical use
- regulate commercial access
- protect derived systems and AI integrations

---

## 2. Licensing Layers

### 2.1 Zenodo Releases (Scientific Layer)
- License: CC BY-NC (or specified per version)
- Purpose: academic sharing, research visibility
- Restriction: no commercial use without permission

---

### 2.2 GitHub Repository (Technical Layer)

#### Core Code / Engine (planned or future)
- License: AGPLv3 (or equivalent copyleft)
- Requirement: derivative systems must remain open-source or obtain commercial license

#### Legal Layer (legal/ directory)
- Custom Akimov License Framework (ALF)
- Defines:
  - derivative works
  - interaction rules
  - enforcement mechanisms

---

### 2.3 Derived Systems and AI Models

Any system that:
- uses ToU Engine
- integrates via API / RPC / sockets
- trains models using ToU logic or outputs

is considered a **derivative or combined work** and must:

- comply with AGPLv3 (if open)
OR
- obtain a commercial license from the author

---

## 3. Commercial Licensing

Commercial use includes:

- monetized applications

All such use requires explicit permission and licensing agreement with the author.
